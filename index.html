Fallout Management

Fallout Types

Technical Fallouts

System unavailability, network issues, code bugs

Functional Fallouts

Missing parameters, data issues, validation failures

Fallout Policies

IGNORE Skip error, continue workflow
RETRY Retry with interval/count limits
ROLLBACK Stop and undo all changes
STOP Halt processing
OH Manual intervention via Order Hub

Rollback Process

Automatic compensation when service provisioning fails

Rollback Triggers

P&A Failure

EMS configuration fails → Rollback VC4 reservation

Commit Failure

VC4 commit fails → Delete created P&A services

Inflight Cancel

User cancels order → Undo all completed steps

UNDO Tasks by Service

Create → Delete created resources
Suspend → Resume service
Resume → Suspend service
Change Pkg → Restore previous package
